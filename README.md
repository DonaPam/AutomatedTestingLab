# Лабораторная работа «Автоматизированное тестирование»

## Цель работы
Изучение и применение инструментов автоматизированного тестирования: фреймворка `JUnit` для модульного-тестирования и `Selenium WebDriver` вместе с `TestNG` для UI-тестирования веб-приложений.

## Постановка задач

### Задача 1. Юнит-тестирование библиотеки на Java (JUnit)

#### Дано
- **Библиотечный файл:** `lab-1.0.jar` в соответствии с выданным вариантом.

#### Что требуется сделать
Необходимо разработать проект автоматизированного модульного-тестирования для предоставленной библиотеки.

**Требования к проекту:**

1. **Настройка проекта:**
   - Создать новый Java-проект.
   - В файле сборки `build.gradle/pom.xml` прописать зависимости для фреймворка **JUnit 5**.
   - Добавить файл `lab-1.0.jar` в зависимости проекта ([методчика](materials/JarManual.md)).

2. **Разработка тестов:**
   - Выбрать любые **четыре (4)** метода из конкретного класса библиотеки `lab-1.0.jar` в соответствии с выданным вариантом.
   - Написать юнит-тесты для выбранных методов, используя фреймворк JUnit.
   - **Организация кода:**
     - Все подготовительные операции (инициализация объектов, настройка данных) должны быть вынесены в методы с аннотациями жизненного цикла (например, `@BeforeAll`, `@BeforeEach`).
     - Все завершающие операции (очистка ресурсов) должны быть вынесены в методы с аннотациями (например, `@AfterAll`, `@AfterEach`).
     - Тестовые классы должны наследоваться от базового тестового класса (если требуется общая инициализация).
   - **Параметризация:**
     - Данные для тестов должны передаваться через параметры.
     - Обязательно использовать аннотации `@ParameterizedTest` в сочетании с `@ValueSource` или `@CsvSource`.
     - **Проверить классы эквивалентности и граничные значения** (например, `Double.MAX_VALUE`, отрицательные числа, ноль).

3. **Запуск и отчетность:**
   - Запустить тесты через интегрированную среду разработки (IDE).
   - Продемонстрировать результаты выполнения (зеленые/красные тесты, логи ошибок).
   - Убедиться, что тесты выявляют ошибки в реализации (а они есть в библиотеке).

### Задача 2. UI-тестирование веб-сайта (Selenium WebDriver + TestNG)

- **Браузер:** Google Chrome.
- **Фреймворк:** `Selenium` версии `Selenium 4.15` и `TestNG` версии `7.7` ([методчика](materials/SeleniumManual.md)).
- **Учетные данные для входа (если требуются):**
  - Логин: `Roman`
  - Пароль: `Jdi1234`

#### Что требуется сделать
Необходимо разработать автоматизированные тесты для проверки корректности отображения и функциональности страниц сайта.

**Требования к проекту:**

1. **Настроить окружение:**
   - Подключить зависимости **Selenium WebDriver** и **TestNG** в проект (через `build.gradle` или `pom.xml`).
   - Настроить класс инициализации драйвера (`DriverSetup`):
     - Использовать аннотацию `@BeforeTest` для запуска браузера, настройки окна и перехода на сайт.
     - Использовать аннотацию `@AfterTest` для корректного закрытия браузера (`driver.quit()` или `driver.close()`).

2. **Разработать тестовые сценарии:**
   Создать два отдельных тестовых класса (или метода), соответствующих сценариям из выданного варианта.

4. **Отчетность**: Предоставлен скриншот или лог результатов запуска тестов из IDE/TestNG.
